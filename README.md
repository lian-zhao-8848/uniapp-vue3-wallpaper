# 📝 项目实战复盘：翔翔壁纸（https://static-mp-4c6c8378-1e90-41d1-997d-c0cae6dd7613.next.bspapp.com/#/）

> **项目定位**：一款基于 UniApp 和 Vue 3 (Composition API) 开发的多终端高清壁纸应用，实现了一套代码多端运行的闭环体验。

## 🛠️ 核心技术栈
- **核心框架**：UniApp, Vue 3 (组合式 API)
- **样式方案**：SCSS, CSS 环境变量 (处理安全区域)
- **UI 组件库**：uni-ui
- **异步控制**：Promise, async/await
- **平台支持**：各平台小程序原生 API 

## 📌 核心业务功能
- 壁纸分类展示与瀑布流加载
- 首页 Banner 海报轮播
- 高清大图全屏滑动预览
- 壁纸本地评分与状态反馈
- 客户端图片一键下载及社交分享

---

## 🚀 核心职责与技术沉淀

### 1. 组件化架构与定制开发
- **通用组件抽离**：剖析页面业务结构，成功抽离出公共标题、评分弹窗、专题模块等通用性组件，降低代码耦合度。
- **数据流与通信**：严格使用 `Props` 进行数据类型校验，通过自定义事件 (`emit`) 处理组件交互回调。
- **高阶定制**：灵活运用**具名插槽 (Slots)**，让同一组件在不同页面呈现定制化排版，大幅提升组件的复用率。

### 2. 网络请求封装与拦截优化
- **Promise 二次封装**：脱离基础调用，基于 `Promise` 对底层的 `uni.request` 进行了统一的模块化封装。
- **请求拦截器配置**：在封装层统一处理公共请求头（如全局注入 `access-key`）、状态码校验及异常捕获 (`try-catch`)。
- **防抖与节流**：配合业务逻辑，有效拦截并阻止了滚动触底时可能产生的频繁无效请求，提升了接口的稳定性和服务器性能。

### 3. 极致性能与首屏渲染优化
- **渲染痛点解决**：针对壁纸类应用“大量图片并发渲染”导致卡顿的痛点，实施了多维度的优化策略。
- **渐进式加载**：综合运用**图片懒加载 (lazy-load)** 与**触底加载更多**机制。
- **视觉过渡**：引入**骨架屏 (Skeleton)** 技术填补数据请求期的白屏时间。通过以上方案，有效避免了首次加载时的多余网络消耗，首屏可见时间（FCP）显著提升。

### 4. 原生系统能力与离线缓存策略
- **系统级适配**：调用客户端底层原生 API 获取系统信息，精准计算并处理刘海屏状态栏、胶囊按钮及底部安全区域 (`safe-area-inset-bottom`) 的 CSS 适配。
- **授权与端能力**：严谨处理用户隐私授权逻辑 (`openSetting`)，实现保存图片到系统相册及拉起微信/原生转发分享功能。
- **本地持久化**：利用 `StorageSync` 打造本地缓存方案，对用户的“搜索历史”和“已评分状态”进行离线化存储，极大优化了在无网或地铁等弱网环境下的产品体验。
